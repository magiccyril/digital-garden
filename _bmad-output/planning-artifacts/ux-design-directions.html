Le paradigme @theme de Tailwind v4 expliqué

Tailwind v4 introduit une nouvelle façon de définir les tokens de design : le bloc @theme dans le CSS. Ce changement est fondamental car il aligne Tailwind avec les CSS custom properties natives.

Les tokens définis dans @theme deviennent automatiquement des classes utilitaires Tailwind ET des variables CSS accessibles partout.

Contrairement à Tailwind v3 où la configuration se faisait en JavaScript, v4 fonctionne entièrement en CSS. Cela signifie que votre design system est maintenant consultable dans les DevTools sans avoir à lire un fichier de config.

Un lien vers une note qui n'existe pas encore ressemble à cette note sur les CSS layers — visuellement distinct pour indiquer que l'idée n'est pas encore développée.
